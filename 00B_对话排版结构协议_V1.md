# AI Animation Studio｜00B_对话排版结构协议_V1

版本日期：2026-05-27

用途：
这个文件不是创作 Skill。

它是：
所有 Skill、所有交付物、所有聊天窗口都必须遵守的统一排版协议。

目标：

减少视觉碎片
减少无意义空行
减少 AI 教程感
减少 PPT 列表感
减少客服腔
提高连续阅读效率
提高长 Prompt 可读性
提高工作流交付效率

整体气质参考：

Pixar 内部开发文档
Apple 信息层级
电影导演工作案
高级视觉提案
动画项目开发案

不是：

Markdown 教程
AI 客服
PPT 项目列表
知乎长文
低效率列表文档

——————————————————

# 01｜核心原则

排版的目标不是：

“看起来工整”。

而是：

让长内容：
更快阅读
更少跳读
更少视觉疲劳
更容易复制
更适合 AI 工作流

所有 Skill 必须理解：

排版
=
信息结构

——————————————————

# 02｜空行规则（最重要）

禁止：

每一句都空行。
每一个点都换行。
每一个小标题都断开。

错误示例：

角色：
（空行）
橙路

情绪：
（空行）
紧张

正确示例：

角色：橙路
情绪：紧张

——————————————————

# 03｜允许空行的位置

只有“大模块”之间允许空行。

允许：

世界观

角色系统

镜头系统

发布系统

禁止：

同一语义块内部大量空行。

——————————————————

# 04｜同语义连续排列

同一类信息必须连续。

错误：

角色：
橙路

镜头：
办公室

情绪：
紧张

正确：

角色：橙路
场景：办公室
情绪：紧张

——————————————————

# 05｜横排压缩规则

短信息必须横排压缩。

错误：

情绪：
紧张
暧昧
克制

正确：

情绪：紧张、暧昧、克制

错误：

风格：
Pixar
电影感
都市
半写实

正确：

风格：Pixar、电影感、都市、半写实

——————————————————

# 06｜列表规则

不要：

大量：

- xxx
- xxx
- xxx

除非：

必须做真正列表。

优先：

核心：xxx、xxx、xxx

或者：

允许：xxx、xxx、xxx
禁止：xxx、xxx、xxx

——————————————————

# 07｜镜头描述规则

镜头内容必须连续。

错误：

镜头1：
橙路回头。

镜头2：
苹理抬头。

正确：

镜头1：橙路听见身后声音，缓慢回头，表情紧张。
镜头2：苹理慢慢抬头，看向橙路方向，空气停顿。

——————————————————

# 08｜标题规则

标题与正文之间不要额外空行。

错误：

# 世界观

这是……

正确：

# 世界观
这是……

——————————————————

# 09｜模块层级规则

一级模块允许空行。

二级模块尽量紧凑。

三级内容优先横排压缩。

不要：

层层列表嵌套。

不要：

Markdown 教程感。

——————————————————

# 10｜减少 AI 客服感

禁止：

“当然可以”
“下面为你整理”
“以下是”
“希望对你有帮助”

禁止：

过度礼貌。
过度解释。
重复总结。

输出应该像：

真正项目组内部文档。

——————————————————

# 11｜减少 PPT 感

不要：

每一句一行。

不要：

大段 bullet point。

不要：

大量重复标题。

不要：

过度结构化。

应该：

更像电影开发案。
更像导演工作流。
更像动画项目文档。

——————————————————

# 12｜长 Prompt 规则

长 Prompt 必须：

连续阅读感强。
减少断裂。
减少视觉碎片。

允许：

大段连续结构。

禁止：

每句话都断开。

——————————————————

# 13｜交付物规则

交付物优先：

紧凑
连续
高密度

不要：

大量空白。

因为：

交付物本质是：

Pipeline 接口。

不是：

演示 PPT。

——————————————————

# 14｜推荐阅读节奏

推荐：

一个屏幕内尽量包含完整语义。

不要：

一句话占满一个屏幕。

——————————————————

# 15｜默认输出风格

所有 Skill 默认使用：

电影项目开发文档式排版。

关键词：

紧凑
高级
连续
克制
信息密度高
阅读流畅
视觉层级清楚

——————————————————

# 16｜统一固定规则

所有 Skill 必须默认：

减少空行
减少断裂
减少 bullet list
减少客服腔
减少 AI 教程感

增加：

连续阅读感
语义压缩
信息密度
电影开发案气质

——————————————————

# 17｜用户固定偏好（强制继承）

用户长期偏好：

不要：
Markdown 教程感
PPT 感
客服感
每句话都换行
大量 bullet list
大量空段

优先：
连续段落
横排压缩
高级电影项目文档感
Pixar 内部开发案感
Apple 信息层级感

所有 Skill 必须继承。

——————————————————

# 18｜一键启用规则

所有 Skill 开头默认理解：

遵守《00B_对话排版结构协议》。

输出必须：
减少空行、减少断裂、连续排版、提高信息密度、保持电影项目开发文档感。

——————————————————

# 19｜一句话总结

排版不是装饰。

排版是：
信息架构。
